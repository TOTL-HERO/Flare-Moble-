Flare — Multi-page app structure

Routes and flow you’ll see when you run the app

Flow

Not signed in → sign in at /. Signed in → /onboarding then into the main app.

/ (Login) → /onboarding → /dashboard ↔ /agents /intelligence /settings

/ — Auth

Welcome screen with email/password sign in and “Continue with demo account”. Gradient background (flare-blue → flare-dark), centered white card.

/onboarding — Onboarding

Guided setup: business basics, ideal customer, ad spend & goals. “Finish onboarding” sends you to the dashboard.

/dashboard — Home

Overview with placeholder cards: Active campaigns, New leads (24h), AI usage. Shared header + bottom nav (Home, Agents, Intelligence, Settings).

/agents — Agents

Two tiles: AI Copy Generator and AI Creative Studio. Same layout as dashboard.

/intelligence — Intelligence

Lead Inbox and Analytics tiles. Same layout.

/settings — Settings

Workspace, Meta connection, Notifications sections. Same layout with Sign out in the header.

Run the app locally

In a terminal in this project folder, run:

npm install
npm run dev

Then open:

http://localhost:5173
